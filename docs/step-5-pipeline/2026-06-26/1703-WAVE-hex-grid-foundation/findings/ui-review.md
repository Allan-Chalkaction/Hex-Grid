## Findings
- **UIR-001** [medium · crit-5 · DEFER] WARNING (governing context): No design-system substrate exists for token-compliance auditing. There is no project CSS/theme source of truth, no .claude/agent-context/ui-review*.md overlay defining the token vocabulary, and no ui-spec-addendum.md for run 1703-WAVE-hex-grid-foundation. Token-compliance (2.1) and design-token-verification (2.2) checks therefore audit against general best practices only. This is expected for a greenfield Wave-1 'working shell' (spec.md scopes W1 to a bare login + map shell with a11y as the only stated visual requirement), but it means no token-namespace finding can be authoritative here. Recommend a ui-spec-addendum + a ui-review agent-context overlay be authored before the first wave that introduces real visual surfaces (forms, data tables, status badges), so subsequent reviews can bind token compliance. DEFER to that wave.
- **UIR-002** [medium · none · DEFER] Hardcoded color values in src/index.css: #1a73e8 (focus outline, line 24), #ddd (borders, lines 46 & 59), #b00020 (.form-error text, line 94), and rgba(255,255,255,0.92/0.95) panel backgrounds (lines 45, 58). In a project WITH a design-token system these are FAIL-class. Here there is provably no token/custom-property layer to reference (index.css defines no :root design tokens beyond font-family/line-height), so there is nothing being bypassed — this is the absence of a design system, not a violation of one. Note also #b00020 on rgba(255,255,255,0.95) is ~7:1 contrast and #1a73e8 focus ring is adequately visible, so no a11y regression. Recommend: when the design system lands (UIR-001), promote these literals to CSS custom properties (e.g. --color-focus, --color-border, --color-error, --surface-overlay) in one pass. DEFER.
- **UIR-003** [low · none · DEFER] Interactive controls have no hover state. The submit button (AuthGate.tsx:104, .auth-gate button[type='submit'] in index.css:86) and the Sign-out button (AuthGate.tsx:64, unstyled .app-header button) rely solely on default UA styling and the global :focus-visible ring — no :hover affordance. Focus-visible IS correctly present app-wide (index.css:23), so keyboard users are served; only the resting->hover transition for pointer users is missing. Acceptable for a bare W1 shell with no design spec, but should be added when the component styling is fleshed out. DEFER.
- **UIR-004** [low · none · DEFER] Minor spacing inconsistency: button padding uses 0.6rem (.auth-gate button, index.css:88) while every other spacing value in the file sits on a 0.25/0.5/1rem rem scale (header padding 0.5rem 1rem, field gap 0.25rem, panel padding 0.5rem 1rem). 0.6rem is the single off-scale value. Also .site-panel top:3rem vs .auth-gate margin 4rem are independent contexts so not a sibling-inconsistency. Trivial; align to 0.5rem or 0.625rem when tokenizing spacing. DEFER.
