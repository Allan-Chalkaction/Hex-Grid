Read `core/skills/shape/SKILL.md` and follow it to run the `/shape <folder-or-file>` per-item interview loop.

`$ARGUMENTS` is the folder or file to shape — a `docs/step-1-ideas/` bucket (typically `needs-shaping/`) in
v1. Walk each not-yet-shaped capture one at a time, halt to riff it, fold the outcome back into the file
(mandatory `## Shaped` section), coarse theme-tag it, and dispose it. Stateless resume skips already-shaped
items by presence-of-section. See the skill for the full loop and disposition rules.
